Protocol: CMQ Binary v1

Frame Header (8 bytes): ┌──────┬─────────┬───────┬────────┬──────────────────┐ │ 0xCA │ version │ flags │ op_type│ payload_len (BE) │ │ 1 by │ 1 by │ 1 by │ 1 by │ 4 bytes │ └──────┴─────────┴───────┴────────┴──────────────────┘ Operations: 0x01 CONNECT 0x02 CONNACK 0x03 PUBLISH 0x04 PUBACK 0x05 SUBSCRIBE 0x06 SUBACK 0x07 UNSUBSCRIBE 0x08 UNSUBACK 0x09 MESSAGE 0x0A PING 0x0B PONG 0x0C DISCONNECT 0x0D ERROR 0x0E INFO

Development Log

2025-05-25
Project kickoff. Analyzed nats-server v2.15.0-dev (~136K LoC). Designed architecture: 1 acceptor thread, N worker event loops, per-connection coroutines with assembly context switch, binary CMQ protocol, arena+slab memory pools. Oracle review incorporated: 8KB default stack with dynamic growth, epoch-based reclamation for sublist, work-stealing between workers.
2025-05-25
Phase 0 complete. 46 files, 2525 lines. 7 test suites (48 tests) all passing. Event loop with epoll/kqueue backends, assembly coroutines (x86_64 + ARM64), arena+slab memory pools, multi-appender logger. Pushed to remote.
2025-05-26
Phase 1 started. Binary protocol parser + subject trie in parallel. Parser: state machine for CMQ binary frames (8-byte header + variable payload). Sublist: trie with * and > wildcards, rwlock thread safety.
2025-05-26
Parser + Sublist complete. 9 test suites (73 total tests) all passing. Parser: 11 tests covering byte-by-byte feed, multi-frame, encode, invalid input. Sublist: 14 tests covering exact match, * and > wildcards, 1K subjects, validation. Next: TCP server acceptor + client lifecycle + PUB/SUB routing.
2025-05-26
Phase 1.2 + 1.4 complete. TCP server + PUB/SUB routing working end-to-end. Server: non-blocking accept, event-driven client handler, INFO/CONNECT/PING/PONG/DISCONNECT. PUB/SUB: subscribe → publish → message delivery verified via integration test. Event loop: level-triggered epoll, eventfd wakeup for clean shutdown. Sublist: now supports multiple subscribers per subject. Bug fix: cmq_frame_hdr_t is 9 bytes (packed), not 8 — caused silent MESSAGE send failure. 10 test suites (77+ tests) all passing.
2025-05-26
Phase 1.5 complete. Configuration file parser implemented. INI-style format: key=value, # comments, [section] headers, quoted strings. Supports: host, port, threads, max_clients, max_payload_size, log settings. 10 tests covering basic, comments, quotes, logging, validation, edge cases. 11 test suites (87+ tests) all passing.
2025-05-26
Phase 2 complete. Queue groups, headers, auth, monitoring. Queue groups: shared subscriptions with round-robin delivery within group. Headers: CMQ_FLAG_HEADERS bit for PUBLISH→MESSAGE header passthrough. Auth: username/password on CONNECT, configurable via config file. Monitoring: atomic u64 stat counters, JSON stats in INFO frame. 12 test suites (92+ tests) all passing.
2025-05-26
Phase 3 complete. Persistence layer implemented. Memstore: ring buffer with seq addressing, truncate, O(1) put/get. Stream: durable named log with consumer tracking, byte-based eviction. Filestore: append-only data + offset index, CRC32 verification, crash recovery. 13 test suites (105+ tests) all passing.
2025-05-26
Phase 4 complete. Full clustering support implemented. Cluster membership: node discovery, heartbeat tracking, state management (joining/active/leaving/offline). Route connections: server-to-server message forwarding with exclude list for loop prevention. Gateway: cross-cluster communication, remote cluster registry, per-cluster connections. Leaf nodes: lightweight edge connections to hub, local subscription tracking, accept/remove leaf clients. Bug fix: filestore tests cleaned stale /tmp data between runs (append mode accumulated state). 14 test suites (117+ tests) all passing.
2025-05-26
Phase 5 complete. Full enterprise feature set implemented. Accounts: multi-tenant isolation with per-account stats, export/import permissions with wildcard matching. TLS: config-driven TLS layer with cert/key/ca/verify settings, server/client session lifecycle. MQTT bridge: topic mapping (MQTT topic ↔ CMQ subject), CONNECT/PUBLISH/SUBSCRIBE/PINGREQ encoding. WebSocket: RFC 6455 frame parser/serializer, mask/unmask, HTTP upgrade handshake with SHA-1 accept key. OpenSSL linked for WebSocket accept key generation. 15 test suites (140+ tests) all passing. All 5 phases complete.
2025-05-26
Examples, README, and documentation complete. 3 example programs: pubsub (server/publisher/subscriber), streaming (memstore/stream/filestore demos), cluster (membership/routing/gateway/leaf). Comprehensive README.md with architecture diagram, protocol spec, module reference, build options. CI covers Linux (gcc/clang), macOS, Windows, ARM64 cross-compile.
